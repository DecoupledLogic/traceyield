# Slice by model, unified with the provider filter

## Goal

Add a model slicing dimension and unify it with the provider filter (E2-F3-S1)
into a single provider+model slicer across the report.

## Definition of Done

- The report can filter by model, and by provider+model together, re-aggregating
  all panels for the selection.
- The `by_model` data already in the payload drives the model options; the
  control degrades gracefully when only one provider/model is present.

## Acceptance Criteria

```gherkin
Scenario: Slice by model
  Given a report with multiple models across providers
  When the user selects a single model
  Then all panels re-aggregate to that model's figures

Scenario: Combined provider and model slicing
  Given both providers with several models each
  When the user selects a provider and a model
  Then the view is scoped to that provider/model intersection and totals reconcile

Scenario: Graceful with one dimension
  Given data from a single provider or single model
  When the slicer renders
  Then the redundant control is hidden or disabled without error
```

## Test Plan

Manual smoke of `report.html` with mixed provider/model data; verify each
selection's totals reconcile against the unfiltered view. Depends on E2-F3-S1
(provider filter) which this unifies with.
