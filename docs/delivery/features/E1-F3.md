# Retention & storage hygiene

## Goal

Keep `usage.db` bounded over time by aging out verbatim raw payloads: the
structural rows stay, but `raw_event.raw` is nulled past a retention window.

## Definition of Done

- A configurable, idempotent age-out pass nulls `raw_event.raw` older than the
  window (default 90 days) without deleting rows.

## Acceptance Criteria

- [ ] Age-out nulls only `raw` on rows older than the window (E1-F3-S1).
- [ ] Structural columns preserved; pass is idempotent and re-runnable.
- [ ] Retention window configurable; boundary behavior tested.

## Test Plan

Seed events straddling the window; assert kept vs. cleared, and that a second run
is a no-op.

## Notes

Blocked on E1-F1, depends on trusting the canonical store. Story: **E1-F3-S1**.
The per-event 32 KB `raw` clamp already ships.
