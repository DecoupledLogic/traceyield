The documentation stops being a flat list of files and becomes a navigable, grouped site — without duplicating any content.

## What changed

- Docs render through an Astro content collection over the canonical markdown in `docs/`, so the site is a *view* of the repo's docs rather than a second copy that can drift.
- A grouped information architecture (`docs-nav.mjs`) drives the side navigation, so related topics — overview, data model, providers — sit together instead of in one long alphabetical list.
- Repo-relative `.md` links in the source are rewritten at build time to site routes (or GitHub source) by the `remark-doc-links` plugin, so links resolve whether you read the file in the repo or on the site.

## Why it matters

There is now one source of truth for the docs and one place to read them. Adding or editing a doc is a single markdown edit; the site picks it up on the next build.
