# Extend coverage and health monitoring to Codex

## Goal

Extend `coverage()`/health monitoring to Codex data so holes and staleness are
caught for both providers.

## Definition of Done

- Health/coverage reconcile the Codex day-series against transcript coverage.
- Codex holes are reported per provider.

## Acceptance Criteria

```gherkin
Scenario: Codex coverage holes are detected
  Given Codex activity on a date with no recorded metrics
  When health monitoring runs
  Then the hole is flagged for the codex provider

Scenario: Both providers appear in health output
  Given Claude and Codex data
  When print_health()/write_health() run
  Then coverage is reported per provider
```

## Test Plan

`coverage(days, scan_dates, today=...)` fixture for `codex`; assert per-provider
health output.
