# raw_event age-out: null out raw payloads older than 90 days

## Goal

Bound the growth of `usage.db` by aging out verbatim raw payloads: periodically
`UPDATE raw_event SET raw=NULL WHERE ts < now − 90d`. The structural rows stay;
only the bulky escape-hatch text is reclaimed. (The per-event 32 KB clamp already
ships.)

## Definition of Done

- An age-out pass nulls `raw_event.raw` for events older than the retention
  window (default 90 days) without deleting the rows.
- Retention window is configurable; the pass is idempotent and safe to re-run.

## Acceptance Criteria

- [ ] Age-out nulls `raw` only for `raw_event` rows older than the window.
- [ ] Structural rows (ids, ts, type) are preserved: only `raw` is cleared.
- [ ] Retention window is configurable and the pass is idempotent.
- [ ] Boundary test: an event at `now−(window−1d)` is kept, `now−(window+1d)` cleared.

## Test Plan

Seed events straddling the window; assert kept vs. cleared and that a second run
is a no-op.
