# Cross-provider report UI

## Goal

Show providers side by side in `report.html` with a cost-by-provider view, a
provider filter, and a neutral token-normalized comparison.

## Definition of Done

- `report.html` renders a cost-by-provider panel and a provider filter.
- A token-normalized comparison view exists.
- Combined dollar totals are clearly labelled as a per-provider projection
  (Decision 0007 D4).

## Acceptance Criteria

- [ ] The report shows a cost-by-provider breakdown and can filter to one provider
- [ ] A token-normalized cross-provider comparison is available
- [ ] Combined dollars are labelled as dependent on per-provider rate cards

## Test Plan

Manual smoke of `report.html` with both providers present; verify the filter
reconciles All vs per-provider figures and the neutral view matches summed
token vectors.
