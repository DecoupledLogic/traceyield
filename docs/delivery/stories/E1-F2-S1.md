# Aggregate flip: regenerate daily/session metrics via GROUP BY over usage.db

## Goal

Stop dual-writing `daily_metrics.json` / `session_metrics.json` from the
transcript parse. Instead regenerate them from a `GROUP BY` over the canonical
`usage.db`, making the SQLite store the single source of truth for aggregates.

## Definition of Done

- Daily + session metrics are produced by SQL aggregation over `usage.db`, not
  by the parallel accumulation path in `report.py`.
- Emitted numbers match the pre-flip values on the real corpus (no regression).

## Acceptance Criteria

- [ ] `daily_metrics.json` derived from a `GROUP BY` over `usage.db`.
- [ ] `session_metrics.json` derived from a `GROUP BY` over `usage.db`.
- [ ] Output equivalent (within rounding) to the current dual-write path.
- [ ] Merge/preservation semantics for rotated-away dates/sessions still hold.

## Test Plan

Assert aggregation query results against the existing fixtures' expected costs;
diff emitted JSON vs. the dual-write path on the real corpus.
