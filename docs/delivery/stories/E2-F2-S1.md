# Un-scope aggregate() to all providers with a by_provider facet, preserving the analyze() oracle

## Goal

Remove the six `provider='claude'` filters, add a provider scope param and a
`by_provider` sub-aggregation, keeping day/session top-level shapes combined.

## Definition of Done

- `aggregate(conn, provider=None)` defaults to all providers; `by_provider` added
  to day/session buckets.
- `TestAggregateEquivalence` is updated to pin `provider='claude'` and stays green.

## Acceptance Criteria

```gherkin
Scenario: All providers are aggregated by default
  Given usage.db holds Claude and Codex turns
  When aggregate(conn) runs with no provider scope
  Then day/session totals sum across both and by_provider has a bucket per provider

Scenario: The Claude oracle is preserved
  Given the same usage.db
  When aggregate(conn, provider='claude') runs
  Then its output equals analyze() turn-for-turn (TestAggregateEquivalence green)

Scenario: Day/session top-level shape is unchanged
  Given a merged daily_metrics.json
  When the new aggregate writes it
  Then no new top-level keys appear beyond by_provider inside each day/session
```

## Test Plan

Mixed-provider fixture; equivalence oracle pinned to `claude`; assert
`by_provider` sums equal the combined total.
