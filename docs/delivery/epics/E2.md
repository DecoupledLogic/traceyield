# Cross-provider reporting

## Goal

Surface every ingested provider (Claude + Codex today, others later) in one
unified report, priced by per-provider rate cards — turning already-captured
cross-provider data into cross-provider insight, on the store E1 delivered.

## Definition of Done

- The report shows Claude and Codex side by side with correct per-provider
  dollars and a neutral token-based comparison.
- Claude's reported numbers are byte-unchanged (the `analyze()` equivalence
  oracle stays green).
- Adding a third provider requires only a rate card + cache entry — no
  structural change.

## Acceptance Criteria

- [ ] Codex usage appears in the report with correct cost, alongside Claude
- [ ] Adding a new provider needs only a rate card + cache entry (no structural change)
- [ ] Existing Claude report output is unchanged (equivalence oracle preserved)

## Test Plan

Full suite green including `TestAggregateEquivalence` pinned to
`provider='claude'`; manual smoke of `report.html` with both providers present in
`usage.db`. Governed by Decision 0007; design in `docs/cross-provider-reporting.md`.
