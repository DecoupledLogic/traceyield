# pricing_history.json gains a provider dimension with backward-compatible reads

## Goal

Nest `record_pricing()` output by provider and read old flat snapshots for
backward compatibility.

## Definition of Done

- New `pricing_history.json` entries are `{date: {provider: {tier: {input, output}}}}`.
- Existing flat entries still parse; the pricing-trend chart still renders.

## Acceptance Criteria

```gherkin
Scenario: New snapshots are provider-nested
  Given today's PRICING with claude and codex
  When record_pricing() runs
  Then the date entry contains a provider -> tier -> {input, output} map

Scenario: Old flat entries remain readable
  Given a pricing_history.json with pre-existing flat {tier: {...}} entries
  When the report loads pricing history
  Then those entries are interpreted as the claude provider without error
```

## Test Plan

`record_pricing(path=...)` fixture asserting the nested shape; a fixture with a
legacy flat entry asserting the backward-compatible read.
