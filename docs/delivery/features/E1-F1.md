# Multi-provider ingest

## Goal

Parse each coding-assistant provider's raw logs into the same neutral `Rec`
stream (`Session`/`Turn`/`ToolCall`/`Segment`/`RawEvent`) so one `usage.db`
holds every provider, keyed by the `provider` column, proving the abstraction:
a new provider is one new class, no schema / `write()` / harness change.

## Definition of Done

- ClaudeProvider (shipped) and CodexProvider both emit the neutral stream into
  one store; re-ingest is idempotent.

## Acceptance Criteria

- [ ] ClaudeProvider ingests `~/.claude/projects` into the neutral stream (done).
- [ ] CodexProvider ingests `~/.codex/sessions` into the neutral stream (E1-F1-S1).
- [ ] Both land in a single `usage.db`, distinguished by `provider`.

## Test Plan

Provider-parametrized fixtures with hand-computable tokens, asserted via SQL;
plus a real-corpus smoke ingest per provider.

## Notes

ClaudeProvider is the reference implementation. Remaining story: **E1-F1-S1
CodexProvider**.
