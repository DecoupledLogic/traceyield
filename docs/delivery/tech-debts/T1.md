# CI/CD GitHub Actions: run test suite on PRs and gate auto-merge on a required check

## Goal

Server-enforced green-tests gate for `cc-usage-analytics`
(`github.com/DecoupledLogic/tokenlens`): a GitHub Actions workflow runs the full
stdlib test suite on every PR and push to `main`, registered as a **required
status check** so `gh pr merge --auto` waits on green CI instead of merging
ungated. Closes the governance gap found in E1-F1-S1 (PR #2 auto-merged with no
checks; the local review-gate was the only barrier). Promoted via `inline:Q20`.

## Definition of Done

- `.github/workflows/ci.yml` runs `python -m unittest test_report test_canonical`
  on `pull_request` and `push` to `main` via `actions/setup-python` тАФ no
  dependency install (stdlib only), no build step.
- The CI job is a **required status check** on `main` branch protection, so
  auto-merge (`gh pr merge --auto`) blocks until it passes.
- Verified end to end: a green run on a normal PR; a deliberately-failing change
  turns the check red and blocks merge.

## Acceptance Criteria

```gherkin
Scenario: CI runs the test suite on a pull request
  Given a PR opened against main
  When the CI workflow triggers on the pull_request event
  Then it checks out the repo, sets up Python, and runs
       "python -m unittest test_report test_canonical"
  And a passing suite reports a green check on the PR
```

```gherkin
Scenario: CI runs on pushes to main
  Given a commit landed on main
  When the CI workflow triggers on the push event
  Then the same test suite runs and its status is recorded on the commit
```

```gherkin
Scenario: A failing test blocks auto-merge
  Given branch protection on main requires the CI check
  And a PR whose change makes a test fail
  When "gh pr merge --auto" is armed on that PR
  Then the required check is red
  And the PR does not auto-merge until the tests are fixed
```

```gherkin
Scenario: No secrets or real user data are needed
  Given the tests parametrize their root=/path= seams to temp dirs
  When CI runs on a clean runner with no ~/.claude or ~/.codex present
  Then the suite passes without secrets, fixtures, or network access
```

## Test Plan

- Push the workflow on a branch; open a PR and confirm the `ci` check appears,
  runs `python -m unittest test_report test_canonical`, and goes green.
- Enable branch protection on `main` requiring the `ci` check; confirm
  `gh pr view` shows it under required checks.
- Open a throwaway PR with an intentionally-broken test; confirm the check goes
  red and `gh pr merge --auto` does not merge until reverted.
- Confirm the runner needs no `pip install` and no `~/.claude`/`~/.codex` data.
