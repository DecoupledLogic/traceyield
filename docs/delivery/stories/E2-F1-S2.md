# Codex rate card and volume-always/dollars-when-priced policy

## Goal

Add a hand-verified Codex rate card and implement the volume-always /
dollars-when-priced policy so recognized-but-unpriced tiers count tokens at $0
(Decision 0007 D3).

## Definition of Done

- `PRICING["codex"]` holds verified OpenAI rates; `CACHE["codex"]` has no write
  tiers.
- A recognized Codex tier with no rate card contributes tokens/msgs but $0 cost.

## Acceptance Criteria

```gherkin
Scenario: Codex turns are priced from the Codex rate card
  Given a Codex turn on a priced tier
  When cost is computed
  Then it uses PRICING["codex"] rates and CACHE["codex"] (cached read discounted, no write premium)

Scenario: A recognized-but-unpriced tier counts volume at $0
  Given a Codex turn whose tier has no PRICING["codex"] row
  When aggregate() runs
  Then the turn adds to tokens and msgs but contributes $0 cost

Scenario: An unrecognized model is still skipped entirely
  Given a turn whose tier is NULL
  When aggregate() runs
  Then the turn is excluded (unchanged from today)
```

## Test Plan

Codex costing fixtures with hand-computed dollars; assert $0-with-tokens for an
unpriced tier and that a `tier`-NULL turn is skipped.
