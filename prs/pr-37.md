The docs site goes from local-only to publicly hosted. It replaces the legacy Jekyll build of `/docs` as the source for the published site.

## What changed

- A GitHub Actions workflow (`deploy-site.yml`) builds `site/` with Node 20 and publishes `site/dist` to GitHub Pages. It triggers on any push to `main` under `site/**` (or the workflow itself), plus manual `workflow_dispatch`.
- The build runs `astro build` followed by `pagefind --site dist`, so the deployed site ships with a client-side search index.
- The workflow uses a least-privilege token (`contents: read`, `pages: write`, `id-token: write`) and a `pages` concurrency group that never cancels an in-flight deploy.

## Why it matters

The project now has a real, shareable home at `https://decoupledlogic.github.io/traceyield/` rather than a `report.html` you open off disk. Everything under `site/public/` is copied verbatim into the published site, so new static sections (like these PR and release pages) publish with no extra deploy plumbing.

## One-time setup

- In the repo settings, Pages must be set to the **GitHub Actions** build type for the deploy step to publish. This is a single manual switch, not something the workflow can do for itself.
