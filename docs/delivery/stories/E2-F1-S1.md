# Per-provider rate cards: PRICING/CACHE/rate_card/cost_of (Claude refactor, behavior-neutral)

## Goal

Introduce nested `PRICING`, per-provider `CACHE`, `rate_card()`, and `cost_of()`,
and route the existing Claude costing through them with zero change to reported
numbers.

## Definition of Done

- `cache_rates()` usage is replaced by `CACHE`/`rate_card()`; `aggregate()`'s cost
  loop calls `cost_of()`.
- All existing tests pass unchanged, including the equivalence oracle.

## Acceptance Criteria

```gherkin
Scenario: Claude cost is unchanged after the refactor
  Given the existing usage.db and PRICING rates
  When aggregate() computes cost via cost_of()
  Then every day and session cost equals the pre-refactor value (TestAggregateEquivalence green)

Scenario: Cache economics come from the provider's CACHE entry
  Given a Claude turn with 5m and 1h cache-write tokens
  When cost_of("claude", tier, ...) is computed
  Then the write tiers are priced at 1.25x and 2x the base input rate

Scenario: rate_card returns None for an unpriced tier
  Given a (provider, tier) with no PRICING entry
  When rate_card(provider, tier) is called
  Then it returns None and cost_of returns 0.0
```

## Test Plan

Unit tests for `rate_card`/`cost_of` with hand-computed values; run
`TestAggregateEquivalence` (pinned to `provider='claude'`).
