A one-bug fix that makes site search actually navigable on the project Pages subpath.

## What changed

- Pagefind records result URLs relative to the indexed `dist` root — i.e. without the `/traceyield` base path — so clicking a result sent you to `/prs/...` instead of `/traceyield/prs/...` and 404'd.
- The search UI's `processResult` hook now prefixes the site base onto any result URL that is root-absolute and does not already carry it. The same fix is applied to anchored sub-results.

## Why it matters

Search went from "finds the page but the link is broken" to actually taking you there. It's the last gap between the docs site being built and being usable on GitHub Pages.
