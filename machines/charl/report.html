How to read this report

Where the money goes — the five token line-items

Each request re-sends the whole conversation so far. To avoid re-billing all of it at full price, the API caches a stable prefix of the prompt. Your tokens split into five lines, each a multiple of the model's base input price:

Fresh input 1×
Brand-new tokens the model reads for the first time (a newly opened file, your latest message). Full input price.
Cache write — 5 min 1.25×
Storing a chunk in the cache the first time costs a 25% premium. It stays reusable for 5 minutes.
Cache write — 1 hour 2×
Same, kept for an hour — double the input price to write. Claude Code uses 1h caching, which is why this line is large.
Cache read 0.1×
Reading tokens already in the cache costs a tenth of input price. This is the payoff of caching — but because your ~154K-token context is re-read on every turn, it’s still your single biggest cost line by volume.
Output output rate
Tokens the model generates (its reply + tool calls), billed at the separate, higher output price (Opus $25/1M).

A cached token you reuse costs ~8% of writing it fresh at 1h TTL (0.1 vs 1.25), so caching pays off after ~2 reuses. The risk is invalidation: editing a file or changing tools near the front of the prompt forces an expensive re-write of everything after it.

How to use it to improve

  • Right-size the model. ~98% of spend is Opus. Route routine work (reading, simple edits, exploration) to Sonnet with /model — cheaper input and 0.1× cache reads at $0.20 vs $0.50.
  • Keep context small. Cost ≈ context size × turns; the ~154K/turn re-read is the engine. /clear between tasks so each turn re-reads less.
  • Cut errors = cut wasted turns. Each tool error ≈ one extra turn that re-reads the full context. The per-tool panel’s Est. waste column puts a dollar figure on it. Top offenders: Windows shell errors (use PowerShell for Windows commands) and Write/Edit-before-Read (Read first).
  • Watch the trend, not the day. Switch to Week/Month and step with ←/→ to see whether cost-per-turn and error rate improve after a change.

Glossary

Turn
One assistant response (usually one tool call). “Assistant turns” counts these.
Session
One Claude Code conversation (a distinct sessionId).
Tool error rate
Share of tool results returned as errors. Lower is better.
Est. waste
Modeled error cost = errors × avg cost/call × retry factor. Cost/call is exact; the retry factor (how many extra turns an error triggers) is your tunable assumption.
vs prev
Change from the previous period at the current granularity.
Top sessions
Highest-cost individual conversations across all history. A single session far above the rest is a runaway context — the cheapest thing to fix.
Routing estimate
This period’s Opus tokens recosted at Sonnet/Haiku rates, scaled by your routable-share assumption. An upper bound on /model savings, not a promise.
