# Provider-neutral report branding and copy

## Goal

Make the report read as a multi-LLM tool, not a Claude-specific one: neutral
title/copy and no em dashes in the emitted output.

## Definition of Done

- The report title reads "LLM Usage & Health" and Claude-specific copy is
  neutralized or gated on the provider it describes.
- No em dashes appear in the report output.

## Acceptance Criteria

- [ ] The report title is provider-neutral ("LLM Usage & Health")
- [ ] Copy that names a single provider is neutral or scoped to that provider
- [ ] No em dashes appear in the generated report

## Test Plan

Regenerate `report.html` and confirm the title and copy read neutrally with no
em dashes; grep the emitted HTML for the em-dash character.

## Notes

Scope is the **report output** (the `HTML_TMPL` copy in `report.py`), not the
Python source comments. Emerged from operator feedback (2026-07-11) that the
title "TraceYield — Claude Code Usage & Health" (`report.py:948`) reads as
Claude-only. No decision record needed; these are branding/style edits governed
by the E2 cross-provider direction (Decision 0007 D4, neutral framing).
