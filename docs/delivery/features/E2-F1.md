# Per-provider pricing model

## Goal

Replace the flat, Claude-only `PRICING` table with per-provider rate cards and a
single `cost_of()` function, so cost becomes a pure `(provider, tier)` lookup.

## Definition of Done

- `PRICING` (nested by provider), `CACHE` (per-provider economics), `rate_card()`,
  and `cost_of()` exist and are the one place cost is computed.
- Claude cost is computed through them with identical results.
- Codex has a hand-verified rate card; an unpriced tier costs $0 but still counts
  volume (Decision 0007 D3).

## Acceptance Criteria

- [ ] Cost for every Claude turn is identical before and after the refactor
- [ ] A `(provider, tier)` with no rate card returns $0 without raising
- [ ] Cache economics are read from the provider's `CACHE` entry, not a shared function

## Test Plan

Unit tests on `cost_of`/`rate_card` with hand-computed values; the
`analyze()` equivalence oracle stays green.
